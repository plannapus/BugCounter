How to format the Taxa Name File

The File containing the informations on the taxa needs to be a tab-separated file.
The file should contain at least 10 columns named exactly as follows:
  • Genus
  • GQ i. e. Genus Qualifier: usually a question mark, if needed. Can be empty field.
  • Species
  • SQ i. e. Species Qualifier: can be a question mark, if needed, or a 'group', 'cf' or 'aff' statement. Can be empty field.
  • Subspecies
  • Author: this column should be present but the field itself can be empty.
  • Comment: this column should be present but the field itself can be empty.
  • HigherTaxon: Field used for the grouping of species. Can be a number or a name. Try to avoid really long names (i. e. over 50 characters). Species with an empty field for that category won't be present in the lists!
  • onButton: 'y' if you want this species to be present as a button, 'n' otherwise. Try to limit the number of such species to less than 200.
  • abbreviation: for the species that are to be displayed as buttons, this field gives the desired label on the button. Optimally it should be a string of less than 12 characters.
The file can contain additional columns (for compatibility with RaritasVox) but the 10 columns mentioned here need to be present.

Rare Count mode

The 'Rare Count' mode allows you, after a few tracks have already been counted, to discard the most abundant species from the count.
These species will then be estimated based on the number of specimens per track that were counted during the 'Normal Count' mode phase.
Warning: Those specimens will be added at the end of each new track. As a consequence the Collector's curve will have artificial large steps when this occurs.

Adding a new species during the count

There is the possibility to add a new species during the count by clicking on 'Add a new species'.
Keep in mind that this species will only be added to this count and not to the original Taxa Name File.
The species can be added to the scrollers but not to the buttons.
